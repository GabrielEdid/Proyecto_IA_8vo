# SMS Classification: Ham vs Spam vs Smishing

A Jupyter notebook that classifies SMS messages into three classes — `ham`, `spam`, and
`smishing` — using a classical machine learning workflow (TF-IDF + simple numeric
features) and three interpretable models:

- Complement Naive Bayes
- Logistic Regression
- Linear SVM

## Dataset

The working dataset is built by **reconciling three real, publicly available SMS
datasets** (Section 2 of the notebook), instead of a single synthetic source:

| Source | File | Classes |
|---|---|---|
| Mishra & Soni (2022), Mendeley | `data/raw/Dataset_5971_mishra_soni.csv` | ham / spam / smishing |
| UCI SMS Spam Collection | `data/raw/sms_spam_collection_uci.csv` | ham / spam |
| Combined Smishing (Hosseinpour et al.) | `data/raw/combined_smishing_shaghayegh.csv` | spam / smishing |

The reconciliation harmonizes the schema, normalizes labels, rebuilds the
`URL` / `EMAIL` / `PHONE` indicators with the same regular expressions, removes
duplicates (including cross-source ones), and then keeps **all** available `ham` and
`spam` while capping `smishing` to the `spam` count so no single class dominates.

- Output: `data/Dataset_consolidated.csv`
- Size: 19,417 messages (`ham` 5,133, `spam` 7,142, `smishing` 7,142)
- Columns: `LABEL`, `TEXT`, `URL`, `EMAIL`, `PHONE`

Conventions:
- `data/raw/` holds the **immutable** source files. Nothing writes to it.
- `data/Dataset_consolidated.csv` is **generated** by running the notebook, so it can be
  recreated at any time from `data/raw/`.

> The previous version of the project is kept for reference as
> `notebooks/sms_classification_viejo.ipynb` (+ `.py`), which uses the older single
> dataset `data/Dataset_10191.csv`. Those files and `data/raw/Dataset_10191.csv` are
> not part of the current pipeline and can be deleted if the history is no longer needed.

## Setup

The project is managed with [uv](https://docs.astral.sh/uv/) (Python 3.14):

```bash
uv sync
```

Or with pip inside a virtual environment:

```bash
python3 -m venv env
env/bin/pip install -r requirements.txt
```

## Run

Open `notebooks/sms_classification.ipynb` in Jupyter and run all cells. The first run
regenerates `data/Dataset_consolidated.csv` from the raw sources before training.

From the command line:

```bash
uv run jupyter nbconvert --to notebook --execute --inplace notebooks/sms_classification.ipynb
```

(use `env/bin/jupyter ...` instead if you set up the project with pip).

## Results

On a 20% held-out test set (3,884 messages):

| Model | Accuracy | Macro F1 |
|---|---|---|
| Linear SVM | 0.886 | 0.892 |
| Logistic Regression | 0.883 | 0.889 |
| Complement Naive Bayes | 0.804 | 0.812 |

`ham` is the easiest class to separate; most remaining errors are between `spam` and
`smishing`, which share length, structure, and suspicious vocabulary.
